# Android UI 文案台账

本文件记录农技千问 Android 端用户可能看到的主要文案、触发场景和当前处理建议。它只服务产品文案核对，不是业务规则真源。

## Debug 预览面板复用规则

2026-04-29 的 debug-only 文案样式预览面板验证有效：适合核对“平时不容易触发、但用户可能看到”的隐藏态文案和异常态 UI。以后如果删除后又需要重建，不要只做纯文字清单，应按这个结构恢复：

- 入口只允许出现在 debug 包或隐藏调试入口里，正式 release 不显示。
- 当前临时入口挂在顶部标题“农技千查”的 debug 点击上；右上角按钮已从预览入口释放出来，预留给后续会员中心。
- 面板上半部分是可点条目：每条包含短标题、触发场景 / 文案摘要、选中态。
- 面板下半部分只展示当前选中项，并尽量复用正式组件或同等样式参数，按真实 UI 样式渲染，而不是简单文本。
- 适合预览：黑底输入框浮层、消息尾部失败 / 重试胶囊、AI 免责声明、消息复制菜单、输入框选择菜单、附件底部面板、网络 / 额度 / 图片异常提示。
- 不适合预览：后端系统 prompt、B/C 层摘要 prompt、开发日志、系统权限弹窗、输入法自己的提示。
- 核对完成后可以删除临时入口；若后续再做，仍按“条目列表 + 点选真实样式预览”的模式。

## 主界面常驻

| 文案 | 位置 / 场景 | 当前建议 |
| --- | --- | --- |
| 农技千查 | 顶部标题 | 保留 |
| 欢迎咨询种植 / 病虫害防治、施肥等问题 / 必要时可上传图片 | 空会话欢迎态 | 可后续收短为两行 |
| 描述种植问题 | 输入框 placeholder / overlay placeholder | 保留 |
| 补充作物、部位或症状会更准 | 输入框已有待发送图片时的 placeholder | 保留 |
| 本回答由AI生成，内容仅供参考。 | AI 回复尾部免责声明，靠左小灰字 | 文案先不动，当前只优化样式 |

## 消息状态

| 文案 | 位置 / 场景 | 当前建议 |
| --- | --- | --- |
| 回复未完成 · 点击重试 | assistant 回复中断后，黑底胶囊尾部 | 保留 |
| 发送失败 · 点击重发 | 用户消息发送失败后，黑底胶囊尾部 | 已收口，保留 |
| 网络波动，回复未完成 | streaming 中断原因浮层 | 保留 |
| 今日额度已用完，请明天再试 | 日额度耗尽浮层；后端返回 quota 后当天发送键置灰，点发送 / 重发 / 重试只弹该提示，不新建 AI 消息 | 已收口，保留 |
| 当前请求较多，请稍后重试 | 限流 / 服务忙浮层 | 保留 |
| 本次回复未完成，请重试 | 其他中断浮层 | 保留 |

## 图片相关

| 文案 | 位置 / 场景 | 当前建议 |
| --- | --- | --- |
| 相机 | `+` 附件底部面板入口 | 保留 |
| 照片 | `+` 附件底部面板入口 | 保留 |
| 单次最多4张照片 | `+` 附件底部面板未满 4 张时的提示第一行 | 保留；已满状态不重复显示 |
| 建议拍清病斑、整棵植株、叶背或果实 | `+` 附件底部面板未满 4 张时的提示第二行 | 保留；已满状态不重复显示 |
| 仅支持 JPEG / PNG 格式 | 图片解码失败 | 保留；其他格式若系统可解码会转 JPEG，解码失败统一用此提示 |
| 图片压缩后仍超过 1MB | 理论兜底；正常可解码图片应由 Q60 缩小兜底压到 <=1MB | 保留为异常兜底，正常流程不应出现 |
| 最多4张图片 | 输入框图片数量已满、选择器回传超过剩余槽位、模型调用前图片数量限制、4 张已满时再次打开 `+` 附件卡片 | 保留；照片入口会尽量在系统选择阶段限制，App 和后端继续兜底 |
| 相机打开失败，请重试 | 创建拍照 URI 失败时的浮层 | 保留 |

上传服务未配置、上传响应格式错误、网络超时等上传细节不再进入主界面文案预览；真实发送失败统一落在消息尾部 `发送失败 · 点击重发`，内部错误仍保留在上传链日志 / 兜底回调里方便排障。
多选图片里有部分图片导入失败时，App 静默跳过失败项，只保留可读取图片；带图发送等待期间重复操作图片入口 / 删除图片也静默忽略，不再额外打扰用户。

## 当前不建议做的事

- 不建议把“文案预览”直接塞进正式主界面，避免新增隐藏状态影响当前稳定 UI。
- 若需要 App 内预览，建议做 debug-only 面板，并且只在 debug 构建或隐藏手势下出现；面板必须能点选单条并看到真实样式预览。
- 正式 UI 目前只做必要文案收口，不为了看隐藏文案去制造异常状态。
