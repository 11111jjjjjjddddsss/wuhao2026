# Android UI 文案台账

本文件记录农技千问 Android 端用户可能看到的主要文案、触发场景和当前处理建议。它只服务产品文案核对，不是业务规则真源。

## 主界面常驻

| 文案 | 位置 / 场景 | 当前建议 |
| --- | --- | --- |
| 农技千查 | 顶部标题 | 保留 |
| 欢迎咨询种植 / 病虫害防治、施肥等问题 / 必要时可上传图片 | 空会话欢迎态 | 可后续收短为两行 |
| 描述种植问题 | 输入框 placeholder / overlay placeholder | 保留 |
| AI生成，仅供参考。 | 当前代码仍为“本回答由AI生成，内容仅供参考。”，AI 回复尾部免责声明 | 可后续收短 |

## 消息状态

| 文案 | 位置 / 场景 | 当前建议 |
| --- | --- | --- |
| 回复未完成 / 重试 | assistant 回复中断后，消息尾部 | 保留 |
| 发送失败 / 重发 | 用户消息发送失败后，消息尾部 | 已收口，保留 |
| 网络波动，回复未完成 | streaming 中断原因浮层 | 保留 |
| 今日额度已用完，请明天再试 | 日额度耗尽浮层 | 已收口，保留 |
| 当前请求较多，请稍后重试 | 限流 / 服务忙浮层 | 保留 |
| 本次回复未完成，请重试 | 其他中断浮层 | 保留 |

## 输入框与选择菜单

| 文案 | 位置 / 场景 | 当前建议 |
| --- | --- | --- |
| 已超过6000字，暂时不能发送 | 输入超长浮层 | 可后续收短为“超过6000字，无法发送” |
| 当前网络不可用 | 无网络时发送 / 重试浮层 | 保留 |
| 复制 / 粘贴 / 剪切 / 全选 | 输入框选区菜单 | 保留 |
| 复制 / 全文复制 | 消息选择菜单 | 保留 |

## 图片相关（能力链已存在，正式图片 UI 未完全展开）

| 文案 | 位置 / 场景 | 当前建议 |
| --- | --- | --- |
| 仅支持 JPEG / PNG 格式 | 图片解码失败 | 图片入口上线前再统一核 |
| 图片压缩后仍超过 1MB，请更换更清晰主体、减少无关背景后重试 | 图片压缩后仍超限 | 图片入口上线前再统一核，可能偏长 |
| 未配置上传服务 | 上传服务未配置 | 仅开发 / 配置异常场景 |
| 上传失败：响应格式错误 | 上传接口响应不符合约定 | 图片入口上线前再统一核 |
| 上传异常: ... | 上传异常 | 图片入口上线前再统一核 |
| 图片数量超过限制：最多4张 | 模型调用前图片数量限制 | 后续图片 UI 上线前核 |
| 有图片时必须带文字描述 | 有图无文字时拦截 | 后续图片 UI 上线前核 |

## 当前不建议做的事

- 不建议把“文案预览”直接塞进正式主界面，避免新增隐藏状态影响当前稳定 UI。
- 若需要 App 内预览，建议做 debug-only 面板，并且只在 debug 构建或隐藏手势下出现。
- 正式 UI 目前只做必要文案收口，不为了看隐藏文案去制造异常状态。
