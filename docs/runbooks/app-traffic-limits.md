# App 流量与限制清单

最后核验：2026-06-13，通过代码搜索和 ECS Cloud Assistant 只读检查确认。

本文记录当前会影响 Android App 或生产 API 的流量、速率、大小、时长、次数、UI 窗口等限制，避免上线前误加新的硬闸。

## 当前结论

- Go 后端没有全局 App 用户数、DAU、主聊天总并发硬卡。
- 除非用户明确拍板并理解代价，不新增全局主聊天并发限制。
- 主聊天请求不设置 `max_tokens`，后端不通过模型参数截断输出长度。
- 现有限制主要是防刷、防短信费用异常、防模型成本失控、防上传滥用和保护手机性能。
- 2026-06-13 已把主聊天流 Nginx IP 限制调宽到和普通 API 一样：`60r/m`，burst `80`，并移除单 IP 2 连接限制。

## 线上 Nginx 限制

生产 API 先经过 Nginx，再进入 Go 服务：

| 区域 | 当前线上限制 | 用户影响 |
| --- | --- | --- |
| 普通 API | 每 IP `60r/m`，burst `80` | 登录、快照、App 日志、帮助反馈、会员等都走这里，正常用户一般够用。 |
| 主聊天流 | 每 IP `60r/m`，burst `80` | 已从旧的 `6r/m`、burst `3` 调宽，正常多人共用 WiFi / NAT 不应被 Nginx 挡住。 |
| 上传 | 每 IP `20r/m`，burst `8` | 保护上传带宽和存储，普通单人发图够用。 |
| 连接数 | 上传提交 `/upload` 为 `4`、普通 API 为 `20`，按 IP；主聊天流和图片读取 `/uploads/` 已不再设置 `limit_conn` | 2026-06-13 移除了聊天“同 IP 2 连接”限制；2026-06-21 为模型 / 用户读取图片的 `/uploads/` 单独去掉连接数限制，避免供应商共享出口 IP 拉图被误伤。 |
| 请求体 | `client_max_body_size 2m` | Go 上传仍要求单张 JPEG `<=1MiB`。 |
| SSE 代理 | 聊天 `proxy_read_timeout=600s`，关闭 buffering | 支持长流式回复。 |

如果真机多人共用 WiFi / NAT 仍出现 429，先区分是 Nginx 429、Go 用户级 `20/60s`、会员次数、短信 / 认证还是模型供应商限流；不要通过加全局主聊天并发硬闸、改模型参数或卡输出长度解决。

## Go 后端限制

| 区域 | 当前默认 | 说明 |
| --- | --- | --- |
| 主聊天速率 | 每账号 hash `20` 次 / `60s` | 生产走 Redis；不是全局并发限制。Redis 限流异常时 fail open，避免 Redis 短抖挡住正常问诊。 |
| 主聊天活跃流 | 同一账号同一时间 1 条 | 防重复开模型流和重复成本。 |
| 主聊天最长时长 | `30min` | 防止异常 SSE 一直挂住。 |
| 输入文字 | `6000` 个中文字符 / rune | Android 本地输入框会截住超长粘贴，后端再兜底校验。 |
| 单次图片 | `4` 张 | Android 和后端一致。 |
| 上传文件 | 单张 JPEG `<=1MiB` | Android 上传前压缩，Go 兜底拒绝超限。 |
| 通用 JSON body | `64KiB` | App 日志单独为 `8KiB`；认证、礼品卡等接口还有更小显式限制。 |
| 短信发送 | 手机号+IP `5/10min`，IP `20/10min` | 必须保留，防短信账单被刷。 |
| 短信登录校验 | 手机号+IP `10/10min` | 必须保留，防验证码暴力尝试。 |
| 历史融合认证 token / login | token 每 IP `20/10min`，login 手机号+IP `20/10min` | 仅保留给历史包 / 兼容接口；新 Android 不调用。 |
| App 自动日志 | 每用户/IP 或 preauth/IP `60/10min` | 非业务关键链路；2026-06-13 起 Redis 异常时 fail open，Redis 正常时仍按频率限。 |
| 帮助反馈用户消息 | 每用户/IP `20/10min` | 防反馈刷屏，正常反馈够用。Redis 限流异常时 fail open，避免用户在异常时联系不上客服。 |
| 礼品卡兑换尝试 | 每用户/IP `10/hour` | 防卡码暴力尝试。 |
| 内部 secret 接口 | 每 IP/scope `120/10min` | 保护脚本和内部接口。 |
| 后台登录 | 用户名 hash + IP hash `10/10min`，外层仍有内部接口 IP 保护 | 只保护 `admin.nongjiqiancha.cn` 登录入口，不影响 App 用户登录、聊天或验证码。 |
| 会员每日次数 | Free `6/day`，Plus `25/day`，Pro `40/day` | 产品权益，不是流量限流。 |
| 加油包 | `80` 次 / 包，未用完次数长期保留，可继续购买叠加 | 产品权益。 |
| MySQL 连接池 | max open `10`，max idle `10` | 当前生产环境使用默认值。 |

Redis 限流当前按链路区分：短信、登录、礼品卡、上传、内部 secret 等安全或成本敏感入口保持 fail closed；主聊天、帮助反馈和 App 自动日志这些用户体验 / 排障链路在 Redis 限流操作错误或超时时 fail open，避免 Redis 短抖影响问诊、客服和日志上报。Redis 正常时，上述链路仍按各自频率限流。

## Android 客户端限制

| 区域 | 当前限制 | 说明 |
| --- | --- | --- |
| 聊天 UI 渲染窗口 | 最近 `30` 个用户轮次 | 手机性能边界；不代表后端归档丢失。 |
| 远端历史 hydrate | 后端 `a_rounds_for_ui` 最近 30 轮，并返回 `round_total` | 避免单聊天页无限渲染旧历史；若总轮数更多，Android 顶部用普通用户口径提示更早轮次已保留、后续对话会尽量接上，不暴露后端归档或模型机制。 |
| 输入文字 | `6000` 字 | 本地截住超长粘贴并提示，后端再兜底。 |
| 选择图片 | `4` 张 | 与后端一致。 |
| 原图读取 | 本地 `32MiB` 保护 | 防异常大图拖垮 App。 |
| 上传图片 | JPEG `<=1MiB`，最长边通常 `1024px` | 压缩规则，不是业务过滤。 |
| 远端缩略图 | 单次读取 `2MiB`，12MiB 内存缓存，失败短缓存 `10min` | 保护滑动和内存；过期远端图不反复拉网络。 |
| 普通 Session API | read timeout `60s` | 极端弱网仍依赖恢复链。 |
| 短信发送按钮冷却 | 前端 `60s`，后端另有短信发送限流 | 前端防误点，后端防盗刷；以前端提示为准，不代表验证码只 60 秒有效。 |
| APK 更新下载 | 最大 `200MiB` | 自更新下载安全边界。 |

## 不是内容过滤

这些不属于后端内容过滤：

- 今日农情只要求 JSON 结构和正好 3 条可展示 item；不按养殖水产词、广告词、来源域名、日期、重复标题或摘要字数做内容拦截。
- 用户记忆只清理外层代码块和空白，模型非空输出就整体覆盖写入。
- 主聊天没有本地假流式 / 假回答兜底，也不设置 `max_tokens`。

## 观察项

- Nginx 主聊天每 IP 已调宽到 `60r/m`，burst `80`；如果仍有 429，要结合响应体和日志确认来源，不要再误判成“同 IP 两个人”。
- Redis 限流 fail closed 对短信 / 认证 / 礼品卡 / 上传合理；主聊天、帮助反馈和 App 自动日志已改为 Redis 异常 fail open。
- ECS 固定公网带宽是 `5 Mbps`，这是图片流量和攻击流量的真实容量边界，和代码限流不同。
- 当前 API 前面没有付费 WAF、CDN 或 DDoS 高防；当前保护是安全组、Nginx、Go 限流、阿里云基础 DDoS、云安全中心免费版、SLS 日志和云监控告警。
