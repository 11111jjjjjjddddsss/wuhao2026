# 协议与隐私 Runbook

最后更新：2026-06-15

## 当前状态

设置页“服务协议”当前是本地内置目录页，点开后包含 6 个二级页面：

- 服务协议
- 隐私政策
- 第三方信息共享清单
- 个人信息收集清单
- 应用权限
- 风险提示

这些页面都在 Android 本地渲染，不依赖联网加载，也不跳转外部网页。
App 用户可见正文按正式上线口径写，不展示具体云厂商、模型品牌、系统实现名或供应商账号信息；服务协议正文固定为 6 段，覆盖服务内容、农业建议边界、用户内容与账号权益、会员 / 支付 / 加油包 / 礼品卡、农资信息与交易边界、禁止行为和协议更新。
App 备案已通过，备案号为 `京ICP备2026031728号-2A`；Android 设置页底部以小灰字展示该编号和 `https://beian.miit.gov.cn/` 查询入口，服务协议 / 隐私政策基础信息也同步展示该编号。
App 公安备案仍需单独到全国互联网安全管理服务平台提交；按阿里云备案提示和官方文档口径，网站 / App 通过管局获得备案号后，应在网站 / App 开通 30 日内完成公安联网备案申请。公安备案不是工信部 ICP / App 备案，也不是继续在阿里云备案订单里提交。

官网同时提供正式协议页：

- `https://nongjiqiancha.cn/legal/user-agreement/`
- `https://nongjiqiancha.cn/legal/privacy-policy/`

新包不再集成融合认证 SDK，不再需要 SDK 协议承接页；协议与隐私入口均为 App 内本地页面和官网 HTTPS 页面。

## 登录与隐私同意

- Android 首次隐私同意已合并到登录页协议勾选。
- 本机未记录当前隐私同意版本时，App 直接进入登录页，协议勾选默认未选中。
- 用户勾选“我已阅读并同意《服务协议》《隐私政策》”后，才记录隐私同意、初始化本机业务身份并补报待发送崩溃日志。
- 未勾选时，不请求后端、不初始化 `IdManager`。
- 新包登录页只保留自有普通短信验证码登录入口，不再展示本机号码一键登录，不再拉起第三方 SDK 页面或图形验证。
- 普通短信接口是当前新 Android 登录页主链：发送验证码调用 `/api/auth/sms/send`，登录调用 `/api/auth/sms/login`。

## 当前权限真相

当前 Android 主 manifest 和依赖合并后应包含：

- `INTERNET`
- `ACCESS_NETWORK_STATE`
- `ACCESS_COARSE_LOCATION`
- `ACCESS_FINE_LOCATION`
- `WAKE_LOCK`
- `RECEIVE_BOOT_COMPLETED`
- `FOREGROUND_SERVICE`
- `REQUEST_INSTALL_PACKAGES`
- `com.nongjiqiancha.DYNAMIC_RECEIVER_NOT_EXPORTED_PERMISSION`（应用私有广播保护权限，签名级，不是用户数据读取权限）

当前明确不申请：

- `READ_PHONE_STATE`
- App 相机权限
- 相册 / 存储读写权限
- 录音权限
- 通讯录权限
- 短信读取 / 发送权限
- App 外通知 / 推送权限

`READ_PHONE_STATE / ACCESS_WIFI_STATE / CHANGE_NETWORK_STATE` 已从主 manifest 和融合 SDK 依赖中移除，`scripts/check-android-build-parity.ps1` 会检查 debug / release merged manifest 不含这些权限和融合认证 Activity，并锁定 debug / release packaged manifest 的最终权限白名单。

## 权限用途

- 网络访问：连接后端、上传图片、流式回答、会员、反馈、今日农情、检查更新和短信验证码登录。
- 网络状态：判断网络是否可用，支持登录认证和主聊天联网。
- 定位权限：仅用于用户问诊时把系统定位反查为省、市、区县等地区文本，提高农业建议的地区适配度。App 不上传经纬度，不保存轨迹，不接地图 SDK，不用于广告、推送、统计或线下人员定位。
- WorkManager 相关后台任务和应用私有广播保护权限：用于带图消息在 App 切后台、进程被系统回收或设备重启后按同一条待发送任务有限重试，并保护 App 内部组件调度不被其他应用冒用。
- 安装未知应用相关权限：仅用于用户主动点击“立即更新”后下载 APK 并调起系统安装确认页。

## 图片入口口径

- 照片入口使用 Android 系统 Photo Picker，只访问用户本次主动选择的图片。
- 拍照入口调用外部相机，通过 FileProvider 临时授权写入 App 创建的 URI，不申请 App 相机权限。
- Android Q+ 拍照成功后会把原始照片另存到系统相册 `Pictures/农技千查`，便于用户找回现场照片；输入框、上传和历史展示仍使用 App 私有副本，不额外申请存储权限。

## 当前文案口径

- App 正文不暴露具体云厂商、模型品牌、模型平台名、后台实现名或供应商账号信息。
- 服务协议和隐私政策使用“智能分析服务、云服务、第三方短信服务商、支付服务商”等用户能理解的通用表达。
- 第三方信息共享清单只列当前必要类别；账号登录主线使用平台手机号账号体系和第三方短信服务商发送验证码。
- SLS 业务日志、支付 SDK 等未接入业务主链的能力不能写成已接入事实。
- Redis 只用于手机号认证、主聊天用户级频控、App 自动日志接收、帮助与反馈用户发消息和上传短期限流，不能写成聊天内容、会员、订单或长期资产主链。
- 本版本不包含广告、地图、推送、统计 SDK、支付 SDK、友盟、Bugly、极光或 Firebase。
- App 内服务协议可按正式上线交易口径描述会员、加油包、礼品卡、订单、退款、微信支付 / 支付宝等页面支持的支付方式和售后边界；工程真相仍以 `docs/runbooks/payments.md` 和 `docs/project-state/open-risks.md` 的 R12 为准，真实支付回调、验签、对账和权益发放未接通前，App 不能开放真实扣费入口。
- 当前“删除所有历史对话”只删除问诊聊天历史、A 层滑窗、记忆文档和 30 天归档，不等于完整账号注销或全部个人信息删除。

## 后续巡检清单

每次新增功能前检查：

- 是否新增个人信息类型。
- 是否新增第三方 SDK 或服务商。
- 是否新增 Android 权限。
- 是否改变图片、聊天、反馈、日志或支付数据的保存期限。
- 是否改变删除历史对话、注销账号或数据删除范围。
- 是否需要更新用户协议、隐私政策、风险提示、第三方清单、个人信息清单和应用权限页。
