# 协议与隐私 Runbook

最后更新：2026-05-31

## 当前状态

设置页“服务协议”当前是本地内置目录页，点开后包含 6 个二级页面：

- 用户协议
- 隐私政策
- 第三方信息共享清单
- 个人信息收集清单
- 应用权限
- 风险提示

这些页面都在 Android 本地渲染，不依赖联网加载，也不跳转外部网页。

## 当前权限真相

当前 `AndroidManifest.xml` 只声明：

- `INTERNET`
- `ACCESS_NETWORK_STATE`
- `REQUEST_INSTALL_PACKAGES`

当前不申请：

- 精确定位 / 粗略定位权限
- App 相机权限
- 相册 / 存储读写权限
- 录音权限
- 通讯录权限
- 短信权限
- App 外通知 / 推送权限

图片入口当前口径：

- 照片入口使用 Android 系统 Photo Picker，只访问用户本次主动选择的图片。
- 拍照入口调用外部相机，通过 FileProvider 临时授权写入 App 创建的 URI，不申请 App 相机权限。
- Android Q+ 拍照成功后会把原始照片另存到系统相册 `Pictures/农技千查`，便于用户找回现场照片；输入框、上传和历史展示仍使用 App 私有副本，不额外申请存储权限。

## 当前文案口径

- App 正文不暴露具体模型品牌、模型平台名或供应商账号信息。
- 用户协议和隐私政策使用“第三方大模型和云服务”这类必要委托处理口径。
- 第三方信息共享清单只列当前必要类别；当前使用 ECS / RDS / 第三方大模型服务，账号登录主线会使用手机号认证和短信验证码服务；本机上传链仍在 ECS 本机目录，OSS Bucket、SLS 业务日志、Redis、支付 SDK 等未接入业务主链的能力不能写成已接入事实。
- 本版本不包含广告、地图、推送、统计 SDK、支付 SDK、友盟、Bugly、极光或 Firebase；Android 一键登录客户端 SDK 尚未完成接入前，不写成已可用的一键登录能力。
- 本版本不提供真实支付、自动续费、礼品卡兑换或农资商品交易入口，不产生真实扣费、兑换权益或商品订单；服务范围变化时，必须在使用前明确价格、商品或服务内容、支付方式、退款 / 售后规则、权益生效方式和必要资质信息。
- 当前“删除所有历史对话”只删除问诊聊天历史、A/B/C 记忆和 30 天归档，不等于完整账号注销或全部个人信息删除。

## 后续巡检清单

每次新增功能前检查：

- 是否新增个人信息类型。
- 是否新增第三方 SDK 或服务商。
- 是否新增 Android 权限。
- 是否改变图片、聊天、反馈、日志或支付数据的保存期限。
- 是否改变删除历史对话、注销账号或数据删除范围。
- 是否需要更新用户协议、隐私政策、风险提示、第三方清单、个人信息清单和应用权限页。
