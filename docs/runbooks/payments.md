# 支付与会员订单 Runbook

最后更新：2026-05-22

## 目的

记录农技千查后续接入真实支付时的边界、流程和风险，避免把当前开发期会员接口误当正式支付系统。

当前未接入任何真实支付 SDK、支付渠道、自动续费或退款流程。

## 当前代码真相

- Android 会员中心和设置页会员中心里的开通 Plus、开通 Pro、升级 Pro、购买加油包按钮都只提示“支付功能暂不可用”。
- Android 不调用 `/api/tier/renew_plus`、`/api/tier/renew_pro`、`/api/tier/upgrade_plus_to_pro` 或 `/api/topup/buy`。
- 后端这 4 个接口仍存在，但只是本地 / 内测开发期直改接口。
- 开发期订单接口默认返回 `PAYMENT_NOT_CONFIGURED`。
- 只有显式设置 `ALLOW_DEV_ORDER_ENDPOINTS=true` 且当前环境不是 `APP_ENV / ENV / GO_ENV = prod / production` 时，开发期订单接口才会放行。
- 生产环境即使误设 `ALLOW_DEV_ORDER_ENDPOINTS=true`，只要 `APP_ENV / ENV / GO_ENV` 是 `prod / production`，后端也会强制关闭开发期订单接口。
- 当前 `orders` 表只记录开发期成功结果，字段包括 `order_id / user_id / type / amount / created_at / status / result_json`；它不是正式支付订单表。

## 现有开发期接口

- `POST /api/tier/renew_plus`：开发期续 Plus。
- `POST /api/tier/renew_pro`：开发期续 Pro。
- `POST /api/tier/upgrade_plus_to_pro`：开发期 Plus 升 Pro，并计算升级补偿。
- `POST /api/topup/buy`：开发期购买加油包，仍要求当前有效档位为 Plus / Pro。

这些接口只允许本地 / 内测调试，不允许公开生产使用。

## 正式支付必须改成什么

正式支付接入后，会员权益发放必须收敛到服务端支付回调 / 对账，不由 Android 直接改权益。

推荐主链：

1. Android 发起购买请求，请求后端创建待支付订单。
2. 后端生成支付渠道所需参数，返回给 Android 调起支付。
3. 支付渠道异步通知后端。
4. 后端验签、校验订单号、金额、商品类型、用户、支付状态和幂等键。
5. 后端在事务内发放会员 / 加油包 / 升级补偿，并写正式订单流水。
6. Android 只轮询或刷新 `/api/me` 展示结果，不自己判断支付成功。

## 正式订单表至少需要

- 商户订单号。
- 支付渠道订单号。
- 用户 ID。
- 商品类型：Plus、Pro、Plus 升 Pro、加油包。
- 应付金额、实付金额、币种。
- 订单状态：pending、paid、failed、closed、refunded。
- 支付渠道：微信、支付宝或后续渠道。
- 创建时间、支付时间、回调时间、更新时间。
- 原始回调摘要或受控脱敏后的回调内容。
- 幂等发放状态和发放结果。

## 必须防住的坑

- 不能信任 Android 上报的“支付成功”。
- 不能只凭前端订单号发权益。
- 不能金额不校验、商品不校验、用户不校验。
- 支付回调可能重复到达，必须幂等。
- 支付回调和用户刷新 `/api/me` 是异步关系，Android 要允许短暂“支付处理中”。
- 退款、撤销、失败和超时关闭不能和成功订单混在一起。
- 自动续费如果未来接入，必须单独做签约、解约、续费通知、扣款失败、到期和协议文案；当前产品没有自动续费。
- 支付密钥、商户私钥、平台证书、APIv3 密钥等只能放服务端密钥管理或环境变量，不能进 APK，不能写仓库。

## 买服务器后必须补

- 选定支付渠道和商户主体。
- 配置支付回调公网 HTTPS 地址。
- 新增正式订单表和支付回调处理接口。
- 接入支付渠道签名 / 验签 / 回调解密能力。
- 新增支付创建接口、支付状态查询接口和支付结果轮询。
- 会员 / 加油包权益发放只从服务端已验签支付结果触发。
- 管理后台补订单查询、退款 / 对账状态、人工补偿和审计。
- 更新用户协议、隐私政策、风险提示、第三方信息共享清单和应用商店材料。

## 上线观察指标

- 创建订单失败率。
- 支付回调验签失败率。
- 支付成功但权益未发放。
- 重复回调命中幂等。
- 用户支付后 `/api/me` 未刷新到新权益。
- 退款 / 关闭订单仍显示有效权益。

参考：

- [微信支付 APIv3 签名和验签](https://pay.wechatpay.cn/doc/v3/merchant/4012365342)：微信支付官方说明接口、回调和调起支付都需要签名 / 验签。
- [支付宝开放平台](https://developers.alipay.com/)：支付宝开放能力和移动应用支付接入入口；真实接入时以对应产品文档为准。
