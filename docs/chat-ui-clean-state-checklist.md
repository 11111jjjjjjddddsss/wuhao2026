# Chat UI Clean-State 回归清单

说明：
- 本文件只作为 clean-state 回归参考，不承担主规则职责
- 当前滚动唯一真相以 [AGENTS.md](D:/wuhao/AGENTS.md) 第 6 / 7 节为准

## 目标

确保清除应用数据后：

- 历史可以没有
- `user_id` 可以重置
- 但任何聊天 UI 都不能退回老毛病

## 场景 1：清除数据后首次进入

- 清除应用数据
- 冷启动 app
- 检查聊天页不能白屏、透明、空白异常
- 检查欢迎态和输入区布局正常
- 检查底部输入区不会压住正文区域
- 检查回到底部按钮高度与正常使用时一致，不能因为首帧测量切换突然更高或更低

## 场景 2：清除数据后首次发送

- 输入一条短文本并发送
- 检查用户消息按正常消息流接上
- 检查用户短文本气泡仍然是包裹宽度，不能被重新拉满
- 检查 waiting 小球第一次出现命中工作线附近，用户消息在其上方
- 检查发送首拍只走当前 Compose 发送起步链，不再回退到任何旧 `RecyclerView / scrollToPositionWithOffset / 预测底边保护` 口径
- 检查空列表首发和连续发送共用同一条发送起步链
- 检查生成区底部工作线与输入区避让正常
- 检查免责声明不被输入框遮挡

## 场景 3：清除数据后的流式生成

- 观察标题、加粗、列表、段落在生成中的表现
- 检查不能出现明显飞字、冒头、跳字、重影
- 检查不能出现“先紧后松，再统一重排”
- 检查最后一段不能突然整段蹦出

## 场景 4：清除数据后的手动滚动

- 生成中手动上滑看历史
- 生成中手动下滑但不回到底部
- 检查不会被自动跟随提前抢手
- 只有真正回到底部后，自动跟随才允许重新接管
- 检查底部不能滑出白区、空白页或透明区

## 场景 5：清除数据后首次消息长按复制

- 长按一条静态消息
- 检查系统选区高亮、手柄、自定义黑色 `复制 / 全文复制` 卡片都能正常出现
- 检查消息卡片不会被顶部白遮罩或底部输入区裁掉半截
- 检查极限边界场景下，卡片宁可居中兜底，也不能压到手柄上
- 检查点选区外和按返回键都能退出

## 场景 6：清除数据后首次输入框文本选择

- 在输入框内输入一段文本
- 长按或全选输入框文本
- 检查黑底白字 `复制 / 粘贴 / 剪切 / 全选` 卡片贴在输入框上方
- 检查卡片不和输入框重叠，不飘到顶部标题区
- 检查点输入框外和按返回键都能把卡片收起
- 检查键盘弹起和收起时，卡片仍然跟随输入框位置

## 场景 7：有历史后的重开 app

- 先发送几轮消息，确保本地已有历史
- 关闭 app 再重新打开
- 检查聊天页恢复稳定
- 检查首次重开会自动贴底，并能把底部保留空白露完整
- 检查从后台切回不会强行自动贴底，仍保留用户浏览位置
- 检查免责声明、正文底部、输入区避让正常
- 检查没有明显整块跳动、白屏、到底部失败

## 失败判定

出现以下任一情况，视为 clean-state 回归失败：

- 清除数据后 UI 看起来像“回到老版本”
- 只能依赖旧历史或旧视口缓存才正常
- 底部正文或免责声明被输入框遮挡
- 输入框选择菜单重新回退成顶部白色系统工具条
- 用户短文本气泡被重新拉满
- 生成中自动滚动错误抢手
- 生成区出现明显先紧后松、尾段整段蹦出

## 备注

- clean-state 问题属于正式回归，不属于“用户极端操作”。
- 修聊天 UI 时，默认同时验证 clean-state 和有历史两种状态。
