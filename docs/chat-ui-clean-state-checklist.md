# Chat UI Clean-State 回归清单

## 目标

确保清除应用数据后：

- 历史可以没有
- `user_id` 可以重置
- 但任何聊天 UI 都不能退回老毛病

## 场景 1：清除数据后首次进入

- 清除应用数据
- 冷启动 app
- 检查聊天页不能白屏、透明、空白异常
- 检查欢迎态和输入区布局正常
- 检查底部输入区不会压住正文区域

## 场景 2：清除数据后首次发送

- 输入一条短文本并发送
- 检查用户气泡上抬位置正常
- 检查小球锚点稳定
- 检查生成区底部工作线与输入区避让正常
- 检查免责声明不被输入框遮挡

## 场景 3：清除数据后的流式生成

- 观察标题、加粗、列表、段落在生成中的表现
- 检查不能出现明显飞字、冒头、跳字、重影
- 检查不能出现“先紧后松，再统一重排”
- 检查最后一段不能突然整段蹦出

## 场景 4：清除数据后的手动滚动

- 生成中手动上滑看历史
- 生成中手动下滑但不回到底部
- 检查不会被自动跟随提前抢手
- 只有真正回到底部后，自动跟随才允许重新接管
- 检查底部不能滑出白区、空白页或透明区

## 场景 5：有历史后的重开 app

- 先发送几轮消息，确保本地已有历史
- 关闭 app 再重新打开
- 检查聊天页恢复稳定
- 检查免责声明、正文底部、输入区避让正常
- 检查没有明显整块跳动、白屏、到底部失败

## 失败判定

出现以下任一情况，视为 clean-state 回归失败：

- 清除数据后 UI 看起来像“回到老版本”
- 只能依赖旧历史或旧视口缓存才正常
- 底部正文或免责声明被输入框遮挡
- 生成中自动滚动错误抢手
- 生成区出现明显先紧后松、尾段整段蹦出

## 备注

- clean-state 问题属于正式回归，不属于“用户极端操作”。
- 修聊天 UI 时，默认同时验证 clean-state 和有历史两种状态。
