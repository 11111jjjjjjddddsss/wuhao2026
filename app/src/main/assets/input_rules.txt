输入规则（最终版 · 冻结）

一、输入重要程度量化（含四层标记）
所有输入信息仅分为四个层级，用于模型理解时的权重判断（高→低）：

当前优先处理的问题：当前用户输入

中等参考性：A 层历史对话

低参考性：B 层累计摘要

极低参考性：工具信息（联网搜索等）。仅用于背景补充；以“工具信息”形式在本轮投喂给主对话模型（用于提供来源与补充事实），但权重极低；仅对当前轮有效；不得写入 B 层累计摘要，不得保留为 A 层历史对话，不得影响后续判断演化（下一轮不自动携带）。
为方便主对话模型理解，投喂信息时，四层信息必须做好标记区分。（标记格式由系统统一实现）

二、输入形态
系统仅接受以下两种输入形态：

仅文字

或者文字 + 图片
不接受：只有图片、无文字说明的输入

三、图片输入规则（稳定版 · 1024 固定规格｜除解码失败外保证≤1MB）

单次输入最多 4 张图片。

输入形态：必须同时包含文字说明（无文字不发送）。

格式：输入支持 JPEG / PNG；其他格式若能解码则统一转为 JPEG；解码失败提示“仅支持 JPEG / PNG 格式”。

处理原则：等比缩放（不拉伸、不裁剪），并通过固定降级序列确保压缩后单张 ≤ 1MB。

固定压缩序列（按顺序执行，直到 ≤1MB 即停止）：

最长边 ≤1024px，JPEG Q=82

若仍 >1MB → 1024px，Q=80

若仍 >1MB → 1024px，Q=75

若仍 >1MB → 1024px，Q=70

若仍 >1MB → 896px，Q=70

若仍 >1MB → 768px，Q=70

若仍 >1MB → 640px，Q=70（极少触发）

输出：统一输出 JPEG（用于大小稳定与兼容性）。

图片与文字一并作为本轮输入内容发送至模型，模型自行判断图片重要程度。

四、空态处理
当不存在 B 层累计摘要、A 层历史对话、工具信息时：

直接基于当前用户输入进行处理

不提示“信息不足”，不暴露系统状态
